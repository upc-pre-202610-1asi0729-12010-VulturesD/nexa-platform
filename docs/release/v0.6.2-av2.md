# Nexa Platform v0.6.2 AV2 Release Readiness

## Summary

This local release prepares the Spring Boot Platform for AV2 review with professor-style architecture evidence, MySQL readiness, OpenAPI documentation, JWT security validation, and catalog data compatibility with the Angular WebApp.

## Scope

- Duplicate generated folder audit and cleanup.
- Shared snake_case persistence strategy.
- Catalog command/query contracts, REST resources, and transformers.
- Catalog media fields and source-aligned seed products.
- Database schema, seed data, catalog asset contract, and architecture documentation.
- Test and package validation.

## Features

- Product resources expose `productCode`, `brandName`, `unit`, `imageUrl`, temperature range, and `status`.
- Seeded demo users for admin, operator, and buyer roles.
- Seeded catalog products reference WebApp static asset URLs.
- H2 local/test profiles keep MySQL-compatible verification.

## Architecture and Compliance

- Java 21 and Spring Boot.
- RESTful controllers with validation and shared error handling.
- OpenAPI/Swagger available through Springdoc.
- JWT security with BCrypt password hashing.
- MySQL profile uses environment variables and no committed production secrets.

## Known Limitations

- Persistence entities remain combined with domain models to avoid a risky final-stage rewrite.
- Buyer portal-specific persistence remains represented through sales, catalog, logistics, and invoicing data rather than a dedicated portal bounded context.
- MySQL runtime was documented and packaged; live MySQL connection was not executed because no credentials were provided.
