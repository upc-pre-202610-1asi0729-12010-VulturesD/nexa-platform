# Nexa Platform Deployment

Nexa Platform is a Spring Boot REST API deployed separately from Nexa WebApp. Production deployment targets Render with a managed PostgreSQL database.

## Local Verification

Run tests and build the API artifact:

```bash
./mvnw test
./mvnw package
```

Run locally with H2 and seed data:

```bash
SPRING_PROFILES_ACTIVE=local ./mvnw spring-boot:run
```

## Render

`render.yaml` defines:

- Docker web service `nexa-platform`.
- Managed PostgreSQL database `nexa-platform-db`.
- `SPRING_PROFILES_ACTIVE=postgres,seed` for academic demo data.
- `DATABASE_URL` from Render's managed database.
- `JWT_SECRET` generated by Render.
- `JWT_EXPIRATION_MINUTES=120`.
- `JPA_DDL_AUTO=update`.
- `FRONTEND_ORIGIN=https://nexa-webapp.onrender.com`.
- Health check at `/actuator/health`.

The `RenderPostgresDatabaseUrlEnvironmentPostProcessor` converts Render's `postgresql://...` `DATABASE_URL` into Spring JDBC properties before datasource creation. It is registered in `src/main/resources/META-INF/spring.factories`.
